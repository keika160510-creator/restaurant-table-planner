レストランのテーブルプランナー / Restaurant Table Planner
🍽️ レストランの席管理・予約管理を直感的に行えるWebアプリです。
▶ デモ: https://keika160510-creator.github.io/restaurant-table-planner/

概要
テーブルの空き状況をリアルタイムで視覚化し、人数に応じた席の組み合わせ提案から予約・ウォークイン・ウェイティングの管理まで、レストランの現場で必要な席管理機能をひとつにまとめたツールです。

作った背景
レストランでのアルバイト経験を通して、席の管理や案内判断がスタッフの経験と記憶に依存していることに課題を感じました。特に、どの席がいつ空くか・何名をどのテーブルに案内できるかという判断は、忙しい時間帯ほど難しくなりがちです。
そこで、テーブルの使用状況や予約時間を視覚的に整理し、直感的に判断できるツールがあれば現場の負担を軽減できると考え、実際の運用をイメージしながら本ツールを開発しました。

主な機能

席マップ表示: 全テーブルの使用状況を色分けでリアルタイム表示

🔴 使用中 / 🔵 予約あり（未開始）/ 🟡 ウェイティング / ⚪ 空き


自動テーブル提案: 人数を入力すると最適なテーブルの組み合わせを自動提案
3種類の管理: 予約・ウォークイン・ウェイティングをそれぞれ管理
仮想時刻機能: 時間を自由に動かしてシミュレーションが可能
編集・席移動: 登録後でも名前・時間・人数・テーブルを変更可能
日本語 / English 切替: ボタン1つで言語を切り替え


使用技術

HTML — 画面構成
CSS — レイアウト・色分け・モーダル
JavaScript — テーブル提案ロジック・状態管理・UI制御

外部ライブラリは使用していません。単一HTMLファイルで動作します。

工夫した点

テーブルの組み合わせ提案ロジックを独自実装（2名席・4名席・複数テーブル結合など、席の配置に応じたルールを設定）
ウェイティングのお客様が自分自身のテーブルをブロックしてしまうバグを発見・修正するなど、実際に動かしながら改善を繰り返した
仮想時刻と連動してテーブルの表示が自動で切り替わる仕組みを実装


操作方法

お客様の名前・開始時刻・人数・時間・種別を入力
「提案」ボタンを押すと空きテーブルの候補が表示される
候補を選んで「登録」ボタンを押すと席マップに反映される
着席後は「編集」ボタンから情報の変更・テーブル移動が可能
